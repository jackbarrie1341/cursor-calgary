# Finance Buddy 🐈

A budgeting app with a cat that judges your spending. Connect your bank, and a
cartoon cat reacts in real time — happy when you're under budget, sick when
you've blown it — and roasts your habits with a one-liner on the home screen.

The twist: **the roast is generated by a language model running entirely on your
iPhone.** Every other "AI finance" app ships your transactions off to a cloud
LLM. Finance Buddy's spending analysis never leaves the device — it runs on
Apple's on-device Foundation Models (iOS 26), so no AI service ever sees what
you bought.

---

## What makes it interesting

### An on-device financial agent, not a chatbot wrapper

The cat's verdict comes from a real plan-and-act loop on Apple's ~3B on-device
model (`FoundationModels`), not a single prompt. The model is given a toolbox and
decides which tools to call based on your data:

| Tool | What it does |
| --- | --- |
| `getBudgetStatus` | Today / week / month spend, allowance, streak |
| `getRecentTransactions` | Raw recent activity |
| `getMonthlyBreakdown` | Top merchants this month |
| `getRecurringCharges` | **Computes** repeat merchants after normalizing noisy card-network names (`SQ *BLUE BOTTLE #42` ≈ `BLUE BOTTLE LA`) — catches death-by-a-thousand-coffees |
| `getAnomalies` | **Computes** statistical outliers via population z-score against your own spending distribution |
| `getMonthEndProjection` | **Computes** a deterministic run-rate projection and budget pace |

### The math is in Swift, the language is in the model

Small language models are unreliable at arithmetic, so Finance Buddy never asks
the model to do it. All projections, averages, and outlier scores are computed
deterministically in `FinanceCatAnalytics.swift`; the model only decides *which*
analysis matters and how to phrase it. The month-end projection shown to the
user is the Swift-computed number, merged into the model's output after the fact.

### Streaming, structured output

The verdict is defined as a `@Generable` Swift struct, so the framework returns a
typed value (mood, severity, headline, roast, tip) instead of a string to parse.
It's produced with `streamResponse`, so the headline **types out live** in the
cat's speech bubble while it "reads the receipts." If a stream ends incomplete,
the app falls back to a one-shot generation so the user is never stranded.

### Graceful degradation

Availability is checked at every entry point. On devices without Apple
Intelligence the feature degrades cleanly rather than crashing or hanging.

## Architecture

```
┌─────────────────────────┐     ┌──────────────────────────┐
│  iOS app (SwiftUI)       │     │  Backend (Node/Express)  │
│                          │     │                          │
│  • On-device LLM agent   │     │  • Plaid transactionsSync │
│    (FoundationModels)    │◀───▶│    (cursor-based)        │
│  • Deterministic         │ HTTPS│ • Drizzle ORM → Postgres │
│    analytics (Swift)     │     │  • Supabase auth          │
│  • WidgetKit extension   │     │  • Buddy mood engine      │
│  • Realtime cat updates  │◀────│  • Supabase Realtime      │
└─────────────────────────┘     └──────────────────────────┘
```

**iOS** (`finance-buddy/`) — SwiftUI, iOS 18+ (AI features require iOS 26).
On-device LLM agent, deterministic spending analytics, a home-screen WidgetKit
extension, customizable cat, friends/social, and realtime mood updates via
Supabase Postgres change subscriptions.

**Backend** (`backend/`) — Node + TypeScript + Express. Plaid integration with
incremental `transactionsSync`, Drizzle ORM against Supabase Postgres, Supabase
auth middleware, and a budget/mood engine that maps spend-to-allowance ratios
onto the cat's mood and streak.

### Key files

| File | Role |
| --- | --- |
| `finance-buddy/finance-buddy/FinanceCatAgent.swift` | On-device agent: `@Generable` verdict, tool-calling session, streaming |
| `finance-buddy/finance-buddy/FinanceCatAnalytics.swift` | Deterministic recurring/anomaly/projection math |
| `finance-buddy/finance-buddy/FinanceCatVerdict.swift` | Verdict models, snapshot, persistence |
| `finance-buddy/finance-buddy/AppState.swift` | App state, streaming orchestration, realtime sync |
| `backend/src/buddy/engine.ts` | Mood / allowance / streak logic |
| `backend/src/services/transactionSync.ts` | Plaid cursor-based sync |

## A note on privacy

The **spending analysis and roast generation happen entirely on-device** via
Apple Foundation Models — no transaction text is ever sent to an AI service.
Account linking and transaction sync are handled by Plaid and the app's own
backend (standard for any bank-connected app); the on-device guarantee applies
to the AI layer.

## Getting started

### Backend

```bash
cd backend
cp .env.example .env      # fill in Supabase, Plaid Sandbox, ngrok values
npm install
npm run db:push           # push Drizzle schema to Supabase Postgres
npm run dev               # start Express on tsx watch
npm test                  # Vitest unit tests
```

### iOS

Open `finance-buddy/finance-buddy.xcodeproj` in Xcode 26+ and run.

- The app builds and runs on iOS 18+.
- The on-device AI verdict requires **iOS 26 on an Apple Intelligence–capable
  device with Apple Intelligence enabled** (the model does not run in the
  Simulator). On other devices the rest of the app works and the AI feature
  degrades gracefully.

Point `AppConfig.backendBaseURL` at your running backend (ngrok URL for Plaid
webhooks, or a deployed HTTPS endpoint for TestFlight).
